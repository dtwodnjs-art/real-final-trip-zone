수정 코멘트

기존 팀 설계 초안에서 왜 어떤 항목을 고쳤는지, 무엇을 유지했는지 남긴 기록.

Why Changed

주요 수정 이유

01
예시 프로젝트 흔적 제거

TripZone 기준 문서로 재서술하고, 다른 프로젝트 흔적을 제거했다.

02
권한 구조 통일

USER_ROLES, USER_AUTH_PROVIDERS, USER_REFRESH_TOKENS 기준으로 통일했다.

03
문의 구조 교정

게시글+댓글 구조 대신 INQUIRY_ROOMS + INQUIRY_MESSAGES 채팅형 구조로 확정했다.

04
판매자/회원 상태 확장

승인 후 제재와 휴면/차단/탈퇴 같은 운영 상태를 표현할 수 있게 보강했다.

05
예약/결제/리뷰 정합성 보강

취소 추적, 환불 제약, 리뷰 운영 상태, 중복 방지 기준을 강화했다.

06
감사 로그 추가

승인, 차단, 강제 취소, 쿠폰 발급 같은 운영 행위를 추적할 수 있게 했다.

Kept

그대로 유지한 것

  • *_NO 기본키 명명
  • REG_DATE, UPD_DATE 중심 시간 컬럼
  • MEMBER_GRADES, USERS, HOST_PROFILES, LODGINGS, ROOMS, BOOKINGS, PAYMENTS 같은 핵심 테이블명

Service Layer

서비스 레이어에서 막아야 하는 것

  • 관리자 권한 사용자의 실제 승인 처리 검증
  • 중복 예약 방지
  • 숙박 완료 예약만 리뷰 작성 가능하도록 검증
  • 회원 쿠폰 재사용 방지
  • 문의방 접근 권한 검증
